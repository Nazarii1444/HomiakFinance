### 1. User

Сутність User представляє користувача системи та виконує роль акаунта.
Вона зберігає ідентифікаційні та облікові дані, необхідні для автентифікації, персоналізації інтерфейсу та зв’язування користувача з усіма іншими даними системи.

Основні атрибути:
унікальний ідентифікатор користувача;
облікові дані (login, email, password hash);
системні налаштування (часовий пояс, базова валюта).

Сутність User є центральною у системі та пов’язана з усіма бізнес-даними користувача.

### 2. Transaction

Сутність Transaction описує окрему фінансову операцію, виконану користувачем.
Вона використовується для зберігання інформації про рух коштів у системі.

Основні атрибути:

унікальний ідентифікатор транзакції;
посилання на користувача, якому належить транзакція;
сума операції;
дата виконання;
валюта операції.

### 3. Goal

Сутність представляє фінансові цілі, які користувач встановлює для планування витрат або накопичень.
Вона дозволяє аналізувати прогрес досягнення мети на основі транзакцій.

Основні атрибути:
унікальний ідентифікатор цілі;
посилання на користувача;
цільова сума.

Сутність Goal використовується переважно для аналітичних і мотиваційних функцій системи.

### 4. Currency

Довідникова сутність та містить інформацію про валюти, що використовуються в системі.
Вона забезпечує уніфіковане представлення валют та можливість конвертації сум.

Основні атрибути:
унікальний код валюти;
поточний обмінний курс.

### 5. Notification

Відповідає за збереження повідомлень і нагадувань для користувача.
Вона використовується для інформування про дедлайни, фінансові події або досягнення цілей.

Основні атрибути:
унікальний ідентифікатор повідомлення;
посилання на користувача;
текст повідомлення;
дата або час спрацювання.

Дані цієї сутності мають тимчасовий характер і зберігаються до моменту виконання або втрати актуальності.


| Тип даних   | Сутність     | Приклади          | Політика зберігання      |
| ----------- | ------------ | ----------------- | ------------------------ |
| PII         | User         | login, email      | до видалення користувача |
| Security    | User         | password_hash     | постійно                 |
| Financial   | Transaction  | amount, date      | ≥ 5 років                |
| Analytical  | Goal         | target_amount     | до видалення             |
| Operational | Notification | message, deadline | до виконання             |
