Діаграма показує логічну архітектуру проекту, включаючи користувачів, бекенд, бази даних, сховище та допоміжні сервіси. Вона ілюструє потік даних та взаємодію компонентів.

### 1. Шари/рівні

1) Користувачі

Кінцеві користувачі та адміністратори, взаємодіють через веб-браузер.

2) Frontend (React SPA + TypeScript)

Односторінковий додаток (SPA), реалізує UI, роутинг, відправляє запити на API.

3) HTTPS / REST API
   
Безпечний канал передачі даних між Frontend та Backend.

4) Backend (FastAPI + Uvicorn)

Обробляє всі запити, реалізує бізнес-логіку, керує користувачами, рахунками, транзакціями, категоріями та фінансовими цілями.

5) Database (PostgreSQL)

Зберігає структуровані дані користувачів: акаунти, мультивалютні рахунки, транзакції, бюджети, фінансові цілі.

6) Background Tasks / Celery

Фонові процеси, які оновлюють курси валют і надсилають сповіщення.

7) External Services

FX API (Нацбанк України): постачає актуальні курси валют.

Notification Service (Gmail / Push): надсилає сповіщення користувачам.

8) Monitoring / Logging

Збір логів, трасування запитів, метрик продуктивності та помилок.
